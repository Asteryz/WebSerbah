Wat is HTTP?

HTTPHyperText Transfer Protocol is een applicatie protocol dat aan de basis lag van het www. Het www (WorldWideWeb, of ook wel W3, of het Web) is in 1989 begonnen als een project binnen CERN door CERN-lid en wetenschapper Sir Tim Berners-Lee. Het web was en is een systeem bedoeld om documenten uit te wisselen tussen verschillende gebruikers in het publieke netwerk. Deze documenten zijn zo opgemaakt dat deze door de eindgebruikers bekeken kunnen worden met een Web Browser. Nou gaan we het niet over hebben hoe we dit kunnen bekijken (HTML) aangezien dit een ander onderwerp is. Het gaat er hier om hoe deze eindgebruikers met elkaar communiceren. Hoe vragen zij die documenten op? Hoe ziet het protocol eruit? Wat komt er allemaal bij kijken?

Twee eindgebruikers (vaak Server en Client) communiceren met elkaar via HTTP. Globaal gezien stuurt de Client (meestal een Web Browser) een HTTP request naar een Server. De server handelt deze request af en stuurt een response terug naar de Client.

Welke functionaliteit biedt HTTP?

HTTP biedt een uniforme manier voor applicaties om met elkaar te communiceren in een client/server model gebaseerd op stateless requests. Deze manier van communiceren gaat via het zogeheten request-response protocol wat dus HTTP in zijn geheel vormt. De recenste versie van het protocol is HTTP 1.1. Hier zijn wat extra functionaliteiten aan toegevoegd ten opzichte van HTTP 1.0.
Het meest simpele vorm van een HTTP-request is het volgende:

  • De client zendt een GET-request naar de server gebasseerd op een URI.
  • De server ontvangt dit en stuurt een response terug van het document.
  • De server sluit hierbij direct de HTTP-connectie met de client af.
  • De client ontvangt de response en verwerkt dit.
